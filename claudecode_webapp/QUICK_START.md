# 🚀 クイックスタートガイド

## インストール & 起動

```bash
# プロジェクトディレクトリへ移動
cd ~/Desktop/claude-file-uploader

# 依存関係のインストール
npm install

# サーバー起動
npm start
```

アプリケーションが `http://localhost:3000` で起動します。

## 使用方法

### 1️⃣ ファイルアップロード
- **方法1**: ドラッグ&ドロップでファイルをアップロードエリアに置く
- **方法2**: アップロードエリアをクリックしてファイルを選択

### 2️⃣ 質問/リクエスト入力（オプション）
```
例：
- "このファイルのエラーを分析してください"
- "このコードの機能を要約してください"
- "重要な情報を抽出してください"
```
入力しない場合は、デフォルトの一般分析が実行されます。

### 3️⃣ 分析実行
「ファイルを分析」ボタンをクリック → 結果が表示されます

## 結果表示

分析完了後、以下の情報がカード型で表示されます：

| カード | 内容 |
|------|------|
| **摘要** | Claude による1-2文の要約 |
| **メトリクス** | 複雑度、行数、言語、可読性(0-100%)、保守性(0-100%) |
| **主な発見** | 重大度別(🔴HIGH/🟡MEDIUM/🟢LOW)カラーコード付き |
| **推奨事項** | 優先度別(🔴HIGH/🟡MEDIUM/🟢LOW)カラーコード付き |
| **コードスニペット** | Prism.js による構文ハイライト |
| **詳細分析** | 完全な分析テキスト |

## エクスポート機能

結果ページの上部に3つのボタンがあります：

### 📥 PDF
- 分析結果を A4 PDF ファイルにエクスポート
- `{ファイル名}_analysis.pdf` として保存

### 📥 JSON
- 構造化分析データを JSON 形式でダウンロード
- `{ファイル名}_analysis.json` として保存
- 他システムとのデータ連携に便利

### 📋 コピー
- フォーマット済みテキストをクリップボードにコピー
- メール、ドキュメント等に直接ペースト可能

## 色分けコード

### 発見事項の重大度
- 🔴 **HIGH** (赤): 重要な問題
- 🟡 **MEDIUM** (黄): 中程度の問題
- 🟢 **LOW** (緑): 軽微な問題

### 推奨事項の優先度
- 🔴 **HIGH** (赤): 優先的に対応
- 🟡 **MEDIUM** (黄): 中程度の優先度
- 🟢 **LOW** (緑): 低優先度

## トラブルシューティング

### エラー: "No file uploaded"
→ ファイルが正しく選択されているか確認してください

### エラー: "Bedrock client not initialized"
→ AWS 認証情報が設定されているか確認してください
→ `.claude/settings.json` に AWS クレデンシャルが存在するか確認

### 分析結果が表示されない
→ ブラウザのコンソール（F12）でエラーが出ていないか確認
→ インターネット接続を確認
→ Claude API/AWS Bedrock の状態確認

### PDF/JSON エクスポートが機能しない
→ 広告ブロッカーが CDN ライブラリをブロックしていないか確認
→ ブラウザをリロード

## よくある質問

**Q: 複数ファイルを一度に分析できますか？**
A: 現在は単一ファイルの分析に対応しています。複数ファイル分析は今後の追加予定です。

**Q: 分析履歴は保存されますか？**
A: 現在はセッション中の分析結果のみ表示されます。履歴保存は将来の機能予定です。

**Q: オフラインで使用できますか？**
A: No。Claude API/AWS Bedrock への接続が必須です。

**Q: プライベートなコードを分析できますか？**
A: はい。ファイルはサーバーで一時的に処理された後、削除されます。詳細は README.md を参照。

**Q: 分析の精度は？**
A: Claude モデル (Opus 4.6) の能力に依存します。複雑なコードはより詳細な分析が可能です。
